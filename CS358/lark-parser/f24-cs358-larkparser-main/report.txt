CONFLICTS:
One of the issues I had when initially beginning to test output was when I entered a boolean expression
at the prompt, I got this error: "*** No terminal matches ' ' in the current parser context."
This ended up being because I had a faulty grammar definition, and forgot to include rules to explicitly 
handle the "and" and "or" operators.  Another issue was that I initially forgot to add in the lines at the bottom of
the lark expression that would tell the parser to ignore whitespaces.

After resolving these issues, another issue I ran into was the way the output was being displayed.  Although
the sequence is correct, it also included "orex" and "andex, which I struggled to get rid of until realizing 
I forgot to add the "?" character at the start of each definition.

Additionally, I had trouble with properly printing out the lists.  I found the fix however.  Initially,
my print statements looked like, for example: return ['or', a, b].  This caused a number of issues...
Firstly, it caused my string function to not work at all.  Secondly, it produced this output:
tree.toList() = ['and', Tree('orop', [Tree('truev', []), Tree('notop', [Tree('falsev', [])])]), Tree('truev', [])]

I realized that I forgot to actually visit each node, so I resolved the printing problem by changing it 
to: return ['or', self.visit(a), self.visit(b)]

Another issue I encountered was I did not put brackets around the variables in the return statements for my toList
class, so what actually got produced was a tuple instead of a list, leading to the output not looking the same as the
sample output in the assignment document.
